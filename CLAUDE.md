# CLAUDE.md — 業務週報メーカー 開発メモ

このファイルは、過去の失敗を繰り返さないための記録。
**作業中に指摘・修正・失敗があったら、必ずこのファイルの「失敗と教訓ログ」に追記してからコミットすること。**

## プロジェクト概要

- 毎日の記録メモから「業務週報(人事部用)」を作り、A4縦1枚のPDFに出力する単一HTMLアプリ(`index.html`)
- サーバー不要。ユーザーはファイルをダブルクリックして使う(file://で動くこと)
- 見た目の正解は Excel 原本(`業務週報_YYYYMMDD_古川和幸` シート)。迷ったら原本に合わせる

## ユーザーについて

- 非エンジニア。手順説明は最小限・専門用語なしで
- 成果物(index.html や PDF)は GitHub の説明をするより、**チャットに直接ファイル添付**して渡すのが確実
- 関連ツール: グルメキャリーのスカウト自動生成 Chrome 拡張(fkzyk/gourmetcaree)。AI は Gemini を利用

## 鉄則(過去の失敗から)

1. **リポジトリを混ぜない** — 用途の違うツールを既存リポジトリに入れない。新ツールは専用リポジトリへ
2. **文章の清書はルールベースでやらない** — キーワード抽出では提出レベルにならない。AI(Gemini API)で清書する
3. **罫線はExcel原本と同じ控えめなスタイル** — 全実線のマス目にしない
   - 行動結果の表: 横は点線、縦線は「勤務場所」の両脇のみ。ヘッダー行だけ上下に実線
   - 成果/観察/分析/その他: 行ごとの点線下線のみ(枠・縦線なし)
   - 合計はラベル・枠なしで「40h」とだけ表示(2か所とも)
   - 氏名は枠でなく下線
4. **時間表記は「8h」「40h」形式**(8.0 ではない)。9-18→8h(休憩1h控除)、9-15→6h
5. **PDFは必ずA4縦1枚** — 内容が多いときは自動縮小(printZoom)。変更したら必ずページ数を確認
6. **今週行動予定の平日には既定値(9-18・資さん本社)を自動で入れる**
7. **最新の index.html を変更したら、その場でユーザーにファイル添付で渡す**

## 変更時の検証手順

1. `node --check` でスクリプトの構文確認(`<script>`内を抽出して)
2. サンプルメモで分類・清書ロジックをnodeで単体確認
3. headless chromium でPDF生成し、**ページ数が1であること**と見た目を必ず目視確認:
   ```
   chromium --headless=new --no-sandbox --no-pdf-header-footer \
     --print-to-pdf=out.pdf "file:///path/index.html#d=<base64url(JSON state)>"
   ```
   (`#d=` ハッシュに state のJSONをbase64urlで渡すと任意の内容でレンダリングできる)
4. 日本語フォントがない環境では `fonts-noto-cjk` を入れる
