# CLAUDE.md — 業務週報メーカー 開発メモ

過去の失敗を繰り返さないための記録。**どのセッションでも必ず守ること。**

## このファイルの運用ルール

1. 指摘・修正・失敗があったら、まず「失敗と教訓ログ」に**具体的に**追記してからコミット
2. 同じ種類の教訓が複数たまったら、「鉄則」または「仕様」に**昇格させて集約**する(ログの行は消さない — 細部の経緯はログに残す)
3. つまり: **鉄則・仕様 = 整理された結論 / ログ = 細かい経緯と差分**。両方を常に最新に保つ

## プロジェクト概要

- 毎日の記録メモから「業務週報(人事部用)」を作り、A4縦1枚のPDFをダウンロードさせる**単一HTMLアプリ**(`index.html`)
- サーバー不要。ユーザーはファイルをダブルクリックして使う(file://で動くこと)
- 正解はExcel原本(`業務週報_YYYYMMDD_古川和幸`シート)と提出済みPDF。迷ったら原本に合わせる

## ユーザーについて

- 非エンジニア。手順は番号付きで最小限・専門用語なし
- 成果物はGitHubの説明より**チャットに直接ファイル添付**が確実。index.htmlを変更したら毎回添付して渡す
- 関連: グルメキャリーのスカウト自動生成Chrome拡張(fkzyk/gourmetcaree)。AIはGeminiを利用

## 鉄則

1. **リポジトリを混ぜない** — 用途の違うツールは専用リポジトリへ
2. **文章の清書はルールベースでやらない** — 提出レベルの要約・言い換え・打ち手はAI(Gemini)清書。キーワード抽出は簡易モード扱い
3. **見た目は必ずExcel原本・お手本PDFを確認してから作る** — 勝手な装飾を足さない
4. **保存はワンクリックDL** — 印刷ダイアログを使わせない。PDFは必ずA4縦1枚
5. **変更したら必ず実物のPDFを生成して目視確認**(ページ数・罫線・文字)してから渡す

## 仕様(帳票の見た目・動作)

### 罫線・レイアウト(Excel原本準拠)
- 行動結果の表: 横は点線。縦線は「勤務場所」列の両脇のみ。ヘッダー行だけ上下実線
- 成果/観察/分析/その他: 行ごとの点線下線のみ(枠・縦線・背景色なし)
- 氏名は枠でなく下線。合計はラベル・枠なしで「40h」とだけ表示(先週・今週の2か所)
- フリーライティングは罫線ノート風

### 表記
- 時間は「8h」「6h」「40h」形式(8.0ではない)。9-18→8h(7時間以上は休憩1h控除)、9-15→6h
- 日付は「7/6」形式。既定値: 勤務時間 9-18・勤務場所 資さん本社
- 今週行動予定の平日(月〜金)には既定値を自動で入れる(職務内容は空欄)

### PDF出力
- 「⬇ PDFをダウンロード」ボタン → html2canvas+jsPDF(index.htmlに埋め込み済み・オフライン可)で直接DL
- ファイル名: `業務週報_YYYYMMDD_氏名.pdf`。内容が多いときは自動縮小して必ず1枚に収める
- Ctrl+P の印刷CSSも残してある(フォールバック)

### AI清書(Gemini)
- モデルは gemini-2.5-flash → 2.0 → 1.5 の順にフォールバック。APIキーはlocalStorage保存
- プロンプトには提出済み週報の文例を含め、days/results/observations/analyses/others のJSONを返させる
- APIキーなし用に「指示文コピー→回答JSON貼り戻し」経路もある

## 変更時の検証手順

1. `node --check` で `<script>` 内の構文確認(最後のブロックがアプリ本体)
2. headless chromium でPDFを実生成し、**ページ数=1と見た目を必ず目視確認**:
   `chromium --headless=new --no-sandbox --print-to-pdf=out.pdf "file:///…/index.html#d=<base64url(JSON state)>"`
   (`#d=`ハッシュにstateを渡すと任意内容でレンダリングできる。DL経路は `window.__exportPDF(false)` で検証)
3. 日本語フォントがない環境は `fonts-noto-cjk` を入れる

## 実装上の注意(ハマりどころ)

- index.htmlには html2canvas / jsPDF がインラインで埋め込まれている(約600KB)。**jsPDFのコード内に `</body>` や `<script>` という文字列が含まれる**ため、HTMLを文字列置換で加工するときは `rfind('</body>')` など**最後の出現位置**を使うこと(先頭からのreplaceだとライブラリ内部を壊す)
- CDNはこの開発環境のプロキシで403になることがある。ライブラリはnpmから取得する
